# خارطة طريق نَظْم

هذه الخارطة تقسم العمل إلى مراحل صغيرة قابلة للتحقق. كل مهمة يجب أن تنتهي بتغيير محدود، اختبار مناسب، وتحديث وثائقي عند الحاجة.

آخر تحديث: 2026-05-24، بعد إغلاق مهام المرحلة 0 والتحضير لإصدار `0.1.0`.

- [ ] تشغيل CTest بأسماء ثابتة: ضمان أن أسماء الاختبارات تبدأ بـ `unit_` وأن `ctest -R unit` يلتقطها كلها.

- [ ] حصر الكلمات المفتاحية المدعومة: إنتاج جدول موثق لكل تعليمة عربية وما يقابلها من Opcode.
- [ ] إضافة اختبارات لكل كلمة مفتاحية: اختبار أن كل مدخل في `KEYWORD_TABLE` يُتعرف عليه بدقة.
- [ ] اختبار التعليمات غير المعروفة: ضمان ظهور تشخيص عربي واضح عند استخدام كلمة غير مدعومة.
- [x] تحسين تحليل الفوريّات العربية: دعم الأرقام العربية-الهندية، ASCII، السداسي، الثنائي، والقيم السالبة في lexer.
- [ ] إكمال تحليل الأرقام العربية في كل سياق رقمي: تغطية الإزاحات داخل معاملات الذاكرة بالأرقام العربية-الهندية.
- [ ] اختبار الحدود العددية: حالات الصفر، السالب، أكبر `int64_t`، وأخطاء overflow.
- [x] توحيد الفواصل العربية والإنجليزية: قبول `،` و`,` وتغطية ذلك بالاختبارات.
- [x] دعم التعليقات في المصدر: تجاهل التعليقات والأسطر الفارغة في lexer مع اختبارات.
- [x] اختبار أسماء الوسوم العربية: قبول تعريف الوسم ومرجع الوسم العربيين في lexer.
- [ ] إضافة تشخيص موقع دقيق: التأكد من أن السطر والعمود صحيحان مع UTF-8 متعدد البايتات.
- [ ] توثيق صيغة المصدر في README أو ملف لغة مستقل: إعطاء أمثلة قصيرة للوسوم والتعليمات والمعاملات.

## المرحلة 3: التحليل النحوي وتمثيل التعليمات

- [ ] تعريف AST أو InstructionList بدقة: توثيق ملكية الذاكرة وعدد المعاملات المسموح.
- [x] اختبار تعليمة بلا معاملات: مثل `ارجع` و`لاشيء` و`نداء_نظام`.
- [x] اختبار تعليمة بمعامل واحد: مثل `ادفع ر0` أو قفز إلى وسم.
- [x] اختبار تعليمة بمعاملين: مثل `احمل ر0، ٤٢`.

- [x] كشف فاصلة مفقودة: رسالة خطأ عند غياب الفاصلة بين المعاملات.
- [x] دعم التوجيهات الأساسية: تحليل `.نص` و`.عام` و`.بيانات`.
- [x] إضافة اختبارات تعاف من الأخطاء: سطر خاطئ يليه سطر صحيح ويستمر parser في إنتاج التعليمة الصحيحة.

## المرحلة 4: المرور الأول وجدول الرموز ← CURRENT

- [ ] كشف الوسوم المكررة: `symtable_insert()` يجب أن يعيد خطأ واضحا لا أن يتجاهل التعريف الثاني.
- [ ] اختبار البحث عن وسم موجود: تأكيد أن العنوان المحسوب يعود كما هو.
- [ ] اختبار البحث عن وسم مفقود: تشخيص عربي عند مرجع غير محلول.
- [ ] حساب أحجام التعليمات الثابتة: تأكيد تعليمات مثل `ret` و`nop` و`syscall`.
- [ ] حساب أحجام التعليمات ذات الفوريّات: ضمان اختيار حجم فوري مناسب أو رفض غير المدعوم.
- [ ] معالجة القفزات الأمامية: ضمان أن الوسوم اللاحقة تُحل في المرور الثاني.
- [ ] معالجة القفزات الخلفية: اختبار displacement سالب صحيح.
- [ ] إصلاح تقدير القفز الطويل: استخدام near form أو relaxation حتى لا ينتج ترميز خاطئ بصمت.
- [ ] حفظ رقم السطر مع كل تعليمة: تمهيد لإخراج listing وتشخيصات pass2.
- [ ] توثيق عقد المرور الأول: ما الذي يحسبه وما الذي لا يغيره.

## المرحلة 5: ترميز x86-64 الأساسي

- [ ] توسيع اختبارات REX: تغطية `r8` إلى `r15` في كل مواقع ModRM المهمة.
- [ ] اختبار ModRM للسجل إلى سجل: حالات المصدر والوجهة الأساسية والممتدة.
- [ ] اختبار ModRM للذاكرة: base register، displacement 8-bit، وdisplacement 32-bit.
- [ ] اختبار SIB عند الحاجة: خصوصا `rsp` و`r12` لأنهما يحتاجان SIB حتى في الحالات البسيطة.
- [ ] اختبار ترميز `mov`: سجل/فوري، سجل/سجل، ذاكرة/سجل، وسجل/ذاكرة.
- [ ] اختبار ترميز الحسابيات: `add`, `sub`, `imul`, `idiv`, `inc`, `dec`, و`neg`.
- [ ] اختبار ترميز المنطق: `and`, `or`, `xor`, `not`, وعمليات الإزاحة.
- [ ] اختبار ترميز المقارنة والقفز: `cmp`, `test`, والقفزات الشرطية الأساسية.
- [ ] وضع حد صارم لطول التعليمة: تأكيد عدم تجاوز 15 بايت في كل نتيجة ترميز.
- [ ] تقسيم جدول الترميز عند الحاجة: إبقاء الإضافة سهلة مع نمو عدد التعليمات.

## المرحلة 6: المخرجات الكائنية والربط

- [ ] اختبار relocation: فحص `.rela.text` ميدانيا أو عبر اختبار بايتات.
- [ ] دعم قسم `.data`: إصدار بايتات بيانات منفصلة عن `.text`.
- [ ] اختبار COFF بسيط: إنتاج `.obj` يمكن فحصه بأداة مناسبة أو ربطه لاحقا.
- [ ] توحيد واجهة `output_write`: جعل أخطاء writers عربية وواضحة.
- [ ] إضافة اختبار كتابة ملف: تغطية فشل I/O ونجاح الكتابة دون تلويث مجلدات المصدر.
- [ ] توثيق حدود الربط: الفرق بين إنتاج object file وربط executable.
